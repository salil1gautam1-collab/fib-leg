Trigger timeframe

Entry and Stop are always the zone — entry at the 0.5–0.618 area (the 5m nested fib refines the exact fill), stop a 0.786 close (hedge-covered) — so the Entry and Stop selectors are grayed out. You still choose Exit and Trigger: Square all at T1 takes the whole position off at 0.95 (just below the leg top) — no runner; Let it run + protect scales out ⅓ at 0.95 / 1.272 / 1.618, moves the stop to breakeven after T1, then ratchets the stop up to the previous target as each one hits, so the runner rides while the profit already banked is locked in. Trigger is the interval a close beyond the stop (or a target) must occur on — 5m or 15m — regardless of the detection timeframe. History + stats update so you can A/B exit × trigger.

Setup filter

Every mode enters at the zone (0.5–0.618 + nested fib, 0.786 stop) — this filter just picks which setups you see. All = every zone setup. A+ = only where a broken mountain/valley sits in the zone (old resistance→support lands at your entry — the confluence edge). Only M/W = only setups whose leg origin has a double top/bottom reversal (a W starts the up-impulse we go long on; an M starts the down-impulse we short). Only Pin = only a clean pin bar (hammer / shooting star) at the origin. When a setup has no mountain/valley, it still enters at the plain 0.5–0.618 zone and the chart flags it "⚠ no mountain/valley." If price gaps over the zone or the impulse never retraces to it, there's no clean entry, so the setup is skipped (no chasing — keeps losses small).
